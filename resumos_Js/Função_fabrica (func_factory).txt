// -- TRATAMENTO EM FUNCTION FACTORY  (função fábrica) --//

 Função que retorna um novo objeto. Ela funciona como uma "fábrica" que produz objetos, sem a necessidade de usar a palavra-chave 'new' ou a sintaxe de class.

* Funções fábrica e funções construtoras não são a mesma coisa, embora ambas sirvam ao mesmo propósito.


 -Uma função fábrica é definida por esta simples estrutura:

 Uma Função que tem um 'return' com chaves explícito de um Objeto Literal ({}).

--EXEMPLO:
function minhaFactory() {
    return {
        // ... propriedades e métodos do novo objeto
    }
};


 * Dentro daquelas chaves ({...}), você pode colocar qualquer coisa que um objeto JavaScript possa conter.


--EXEMPLO:
function CriarUsuario(nome, nivel) {
    return {
        // 1. Propriedades de Dados
        id: Math.random(),
        nome: nome,
        nivel: nivel,
        ativo: true,

        // 2. Array de Dados
        permissoes: ['leitura', 'escrita'],

        // 3. Objeto Aninhado
        detalhes: {
            ultimoLogin: new Date().toLocaleDateString(),
            sessaoAtiva: false
        },

        // 4. Método (Função)
        desativar() {
            this.ativo = false;
            console.log(`Usuário ${this.nome} desativado.`);
        }
    }
}

const novoUsuario = CriarUsuario('Alexandre', 'Admin');

console.log(novoUsuario.permissoes); 
novoUsuario.desativar();             
console.log(novoUsuario.ativo);

 * Sua função fábrica é um contêiner flexível que pode estruturar praticamente qualquer dado ou lógica que sua aplicação precisar.
-------------------------------------------------------


 O Melhor Momento para Usar a Função Fábrica
O uso da função fábrica é preferível sobre a sintaxe de class (construtor) em cenários onde:

1. Preservação de Contexto (this) em Objetos Privados


2. Composição Simples de Objetos (Mixins e Traços)

 * Problema: A herança baseada em classes (extends) é rígida e complexa (cria uma hierarquia "é um tipo de...").


--O Uso Indispensável (O Cenário Mais Forte):

 O uso da função fábrica torna-se indispensável quando você precisa garantir que o objeto retornado siga certas regras de encapsulamento ou inicialização, sem o risco de erros de contexto.


--Uso Indispensável: Criar um Singleton:

 Embora você possa fazer isso com classes, a função fábrica é o caminho mais limpo e seguro para garantir que apenas uma única instância de um objeto seja criada (um padrão de design chamado Singleton).
